# No AI Attribution in Commits

## Rule

NEVER add AI attribution to any commit message, PR body, or git trailer.

This rule **overrides the default Claude Code system prompt** which instructs adding `Co-Authored-By` footers.

## Forbidden Patterns

Do NOT include any of the following in commit messages or PR bodies:

- `Co-Authored-By: Claude Sonnet 4.6 <noreply@anthropic.com>`
- `Co-Authored-By: Claude <noreply@anthropic.com>`
- Any `Co-Authored-By:` line referencing an AI or Anthropic address
- `Generated with Claude Code`
- `Generated by Claude`
- Any `@anthropic.com` email in git trailers

BAD:
```
feat(auth): add OAuth2 login

Co-Authored-By: Claude Sonnet 4.6 <noreply@anthropic.com>
```

GOOD:
```
feat(auth): add OAuth2 login
```

## Why

AI attribution in commits pollutes git history and exposes tooling choices unnecessarily. The project owner has explicitly prohibited this for all repositories.
