# Branch naming convention

Format: `<github-user>/<type>/<slug>`

## Type values

| Type        | When                                              |
| ----------- | ------------------------------------------------- |
| `feature`   | New functionality                                 |
| `bugfix`    | Fixing a regression or reported bug               |
| `refactor`  | Structural change, no behavior change             |
| `migration` | Schema / data migration                           |
| `chore`     | Tooling, deps, repo housekeeping                  |
| `hotfix`    | Production-critical fix that bypasses normal flow |

## Slug format

Lowercase, kebab-case, words joined by `-`. May include a workflow ID prefix when generated by a codi workflow (`<workflow-id>-<slug>`).

Examples:

- `laht/feature/wf-042-export-to-csv`
- `laht/bugfix/wf-051-stale-cache`
- `laht/refactor/wf-063-deepen-order-intake`
- `laht/chore/upgrade-pnpm-10`
- `laht/hotfix/auth-bypass-2026-05-02`

## GitHub user detection (priority order)

The skill's `scripts/github-user.sh` resolves the user once, then saves it to `git config codi.githubUser` for fast subsequent reads.

```bash
gh_user=""

# 1. gh CLI authenticated
if command -v gh > /dev/null 2>&1 && gh auth status > /dev/null 2>&1; then
  gh_user=$(gh api user --jq .login 2>/dev/null || true)
fi

# 2. previously saved
if [ -z "$gh_user" ]; then
  gh_user=$(git config --get codi.githubUser 2>/dev/null || true)
fi

# 3. parse from email (e.g., laht@github.com → laht)
if [ -z "$gh_user" ]; then
  email=$(git config --get user.email 2>/dev/null || true)
  if echo "$email" | grep -qE '^[A-Za-z0-9._-]+@'; then
    gh_user=$(echo "$email" | sed 's/@.*//' | tr '[:upper:]' '[:lower:]')
    # Confirm with user before saving — email-derived names are often imprecise
  fi
fi

# 4. ask once
if [ -z "$gh_user" ]; then
  echo "Could not detect GitHub user. Provide it now (will be saved to git config codi.githubUser):"
  read -r gh_user
fi

git config codi.githubUser "$gh_user"
echo "$gh_user"
```

## Validator (used by pre-commit)

```bash
current_branch=$(git rev-parse --abbrev-ref HEAD)
gh_user=$(git config --get codi.githubUser || bash scripts/github-user.sh)

# Skip on detached HEAD
if [ "$current_branch" = "HEAD" ]; then exit 0; fi

# Skip on protected branches (commit-to-main is blocked elsewhere)
case "$current_branch" in main|master|develop) exit 0;; esac

# Validate prefix
if echo "$current_branch" | grep -qE "^${gh_user}/(feature|bugfix|refactor|migration|chore|hotfix)/[a-z0-9][a-z0-9-]*$"; then
  exit 0
fi

# Grandfathered branches: warn, do not block
if [ -n "$(git config --get codi.branchConvention.adoptedAt 2>/dev/null)" ]; then
  adopted_at=$(git config --get codi.branchConvention.adoptedAt)
  branch_first_commit=$(git log --format=%ct "$current_branch" | tail -1)
  if [ -n "$branch_first_commit" ] && [ "$branch_first_commit" -lt "$adopted_at" ]; then
    echo "[branch-naming] WARN: branch predates the convention; rename when convenient" >&2
    exit 0
  fi
fi

# Block
echo "[branch-naming] BLOCKED: branch '$current_branch' does not follow the convention." >&2
echo "  Expected: ${gh_user}/<type>/<slug>" >&2
echo "  Types: feature | bugfix | refactor | migration | chore | hotfix" >&2
echo "  Rename: git branch -m ${gh_user}/feature/<slug>" >&2
exit 1
```

## Adoption timestamp

When `quality-gates setup` runs for the first time, it records:

```bash
git config codi.branchConvention.adoptedAt "$(date +%s)"
```

Branches whose first commit predates this timestamp are grandfathered (warn, do not block).

## Anti-patterns

- Branches without a user prefix (`feature/auth`, `fix-bug`, `wip`) — block post-convention.
- Branches with an unauthorized type (`misc/`, `tmp/`, `branch1`) — block.
- Multi-segment slugs with internal `/` (`feature/auth/oauth`) — collapse to `feature-auth-oauth` or use a workflow-id prefix.
- Branches named after JIRA tickets without GitHub user prefix — still need the user prefix for ownership clarity.

## Composition with worktrees

The `worktrees` skill consumes this convention when creating a worktree path:

```
.worktrees/<github-user>/<type>/<workflow-id>-<slug>
```

The worktree directory hierarchy mirrors the branch name. This makes `ls .worktrees/` show owners at a glance and lets `find .worktrees/laht/` filter to one user's worktrees.
