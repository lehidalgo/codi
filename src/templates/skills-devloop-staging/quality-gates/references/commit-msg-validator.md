# commit-msg validator

Enforces conventional commits + 72-char first-line limit at the `commit-msg` hook stage.

## Allowed types

| Type       | Use for                               |
| ---------- | ------------------------------------- |
| `feat`     | New functionality                     |
| `fix`      | Bug fix                               |
| `docs`     | Documentation only                    |
| `refactor` | Structural change, no behavior change |
| `test`     | Test-only change                      |
| `chore`    | Tooling, deps, repo housekeeping      |
| `perf`     | Performance improvement               |
| `ci`       | CI-related change                     |
| `build`    | Build system change                   |
| `style`    | Code style only (formatter)           |
| `revert`   | Revert a previous commit              |

## Format rules

- Pattern: `^(feat|fix|docs|refactor|test|chore|perf|ci|build|style|revert)(\([^)]+\))?!?: .+`
- Optional scope in parentheses: `feat(auth): ...`
- Optional `!` for breaking change: `feat(api)!: ...`
- First line ≤72 characters.
- Body separated by a blank line if present (not enforced length but encouraged).

## Validator script (`.husky/commit-msg` template)

```bash
#!/bin/sh
# devloop commit-msg validator

msg_file="$1"
[ -z "$msg_file" ] && exit 0

msg=$(head -n 1 "$msg_file" | tr -d '\r')

# Skip merge commits
case "$msg" in
  Merge*|Revert*) exit 0 ;;
esac

types='feat|fix|docs|refactor|test|chore|perf|ci|build|style|revert'

if ! echo "$msg" | grep -qE "^(${types})(\([^)]+\))?!?: .+"; then
  echo "[commit-msg] BLOCKED: invalid commit message format" >&2
  echo "  Expected: type(scope): description" >&2
  echo "  Types: $(echo "$types" | tr '|' ' ')" >&2
  echo "  Example: feat(auth): add OAuth2 login flow" >&2
  echo "  Got: $msg" >&2
  exit 1
fi

if [ "${#msg}" -gt 72 ]; then
  echo "[commit-msg] BLOCKED: first line is ${#msg} chars (max 72)" >&2
  echo "  Message: $msg" >&2
  exit 1
fi

exit 0
```

## Co-author trailers

The devloop CLAUDE.md and global agent rules forbid:

- `Co-Authored-By: Claude` and any model-specific variants
- `Generated by Claude Code` footers
- `noreply@anthropic.com` trailers

The validator does NOT enforce this — it would slow the commit hook with a body scan. Instead, the project's CI runs a separate `forbidden-trailers` check on PRs. See `references/ci-extension.md`.

## Skipping the validator

Forbidden. The skill never runs `--no-verify`. If a commit message must change for legitimate reasons (typo, scope correction), use `git commit --amend` and let the hook re-run.

## Anti-patterns

- Generic types like `update`, `change`, `improve` — not on the allow-list.
- Empty descriptions: `feat:` (nothing after the colon).
- All-caps types: `FEAT:`, `Fix:` — must be lowercase.
- First line wrapping into a paragraph: keep first line ≤72 chars; put detail in the body.
- Co-author trailers from AI assistants — explicitly forbidden by project rules.
