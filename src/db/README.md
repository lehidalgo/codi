# src/db/ — Codi v3 ed.0 schema + migrations

Drizzle ORM target (per ADR-v3ed0-005 + ADR-v3ed0-008).

## Purpose

Schema definitions and migrations for the Codi v3 brain:

- **schema/** (sprint 2) — 11 tables (9 capture/observability + 2 workflow runtime) defined in TypeScript via Drizzle.
- **migrations/** (sprint 2) — generated by `pnpm drizzle-kit generate` from schema changes.
- **client/** (sprint 2) — typed db client wrapper (better-sqlite3 driver in zero, postgres-js in lite/standard/full).
- **seeds/** (sprint 2) — initial data for tests + dev fixtures.

## Schema v1 — 11 tables

### Capture/observability (9)

1. `projects` — project_id, repo_path, git_remote, name, first_seen, last_seen
2. `sessions` — session_id, project_id, agent_type, agent_model, started_at, ended_at, branch, commit_sha, working_dir, transcript_path, workflow_id, total_turns, total_capture_count
3. `prompts` — prompt_id, session_id, turn_no, ts, text, char_count
4. `turns` — turn_id, session_id, turn_no, ts, agent_text (only if trace_level=full), duration_ms, prompt_id
5. `captures` — capture_id, session_id, prompt_id, turn_id, ts, type (10 types), content, raw_marker, file_paths (JSON), workflow_id, phase
6. `tool_calls` — call_id, session_id, turn_id, ts, tool_name, input_json, output_summary, duration_ms, status, error
7. `corrections` — correction_id, session_id, ts, file_path, diff_summary, source_turn_id, detected_via
8. `artifacts_used` — usage_id, session_id, turn_id, ts, artifact_type, artifact_name, event, outcome, duration_ms
9. `_codi_schema_version` — version, applied_at

### Workflow runtime (2)

10. `workflow_runs` — workflow_id, project_id, type, current_phase, status, started_at, ended_at, metadata (JSON)
11. `workflow_events` — event_id, workflow_id, event_type, ts, payload (JSON)

### Indexes + virtual tables (sprint 2)

- FTS5 virtual tables: `captures_fts`, `prompts_fts`
- Standard indexes: `idx_sessions_project_started`, `idx_captures_type_session`, etc.
- sqlite-vec virtual table: `captures_vec` (1536-dim embeddings) — opt-in, deferred
