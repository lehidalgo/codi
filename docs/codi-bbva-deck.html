Safety & Trust Mechanisms

🔍
Drift Detection SHA-256
● Synced ● Drifted ● Missing
Hash-based comparison detects any manual edits to generated files. Run codi status to see the state of all configs at a glance. CI integration blocks PRs with drift.
↩️
Backup & Revert 5 Snapshots
Every codi generate saves a timestamped backup. One command restores any config to a previous state — even after accidental manual edits or corrupted files.
🔐
Verification Tokens Tamper-Proof
Each generated file contains an embedded SHA-256 verification token. AI agents can verify the config came from Codi — not a rogue manual edit. codi verify validates all tokens.
📋
Audit Log Append-Only JSONL
Every generate, revert, and config change is recorded in an append-only JSONL audit log. Full traceability: who changed what, when, from which version — without a separate service.

User Journey 1

The Individual Developer

A developer using Claude Code and Cursor wants consistent AI behavior across both tools — without maintaining two config files.
1
Install & initialize
npm install -g codi-cli && codi init
Codi creates the .codi/ directory with your chosen preset.
2
Choose a preset
codi init --preset balanced
27 rules pre-selected for a balanced developer experience.
3
Generate configs
codi generate
CLAUDE.md and .cursorrules created. Identical standards, both tools.
4
Enable watch mode during dev
codi watch
Edit a rule → saved → both configs updated in 500ms.
5
Commit the source
git add .codi/ && git commit
Config is version-controlled, reviewable, and reproducible.
Best Practices
✅
Commit .codi/, not the generated files
Generated files are build artifacts — add them to .gitignore or commit both. Either way, .codi/ is the source.
✅
Use presets as starting points
Never edit templates directly. Extend them in .codi/rules/ with your project-specific overrides.
✅
Run codi status before PRs
Verify all configs are Synced. Makes config health part of your normal workflow.
