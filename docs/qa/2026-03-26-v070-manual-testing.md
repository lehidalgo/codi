# v0.7.0 Manual QA Testing Guide

**Date**: 2026-03-26
**Document**: 2026-03-26-v070-manual-testing.md
**Version under test**: 0.7.0 (installed from npm)
**Prerequisite**: Node.js >= 20, git-initialized project directory

---

## Purpose

This guide walks a human tester through every interactive and observable feature of Codi v0.7.0. Each step includes the exact command to run, what to look for, and a checkbox to mark pass/fail. Steps marked **HUMAN** require terminal interaction or visual inspection. Steps marked **VERIFY** are observation checks after a human action.

---

## Pre-Test Setup

### Step 0.1 — Clean slate

Remove any previous Codi installation and generated files from the test project.

```bash
# If codi is already installed and initialized:
npx codi clean --all --force

# Or manually:
rm -rf .codi/ CLAUDE.md AGENTS.md .cursorrules .windsurfrules .clinerules
rm -rf .claude/ .cursor/ .cline/ .windsurf/ .agents/ .codex/
```

| Check | Expected | Status |
|-------|----------|--------|
| No `.codi/` directory exists | `ls .codi/` returns "No such file" | [ ] |
| No generated files exist | `ls CLAUDE.md .cursorrules AGENTS.md .windsurfrules .clinerules 2>&1` all missing | [ ] |

### Step 0.2 — Install from npm

```bash
npm install -D codi-cli@0.7.0
```

| Check | Expected | Status |
|-------|----------|--------|
| Install succeeds | No errors | [ ] |
| Version is correct | `npx codi --version` outputs `0.7.0` | [ ] |
| Help works | `npx codi --help` lists all commands | [ ] |

## Phase 1: Init Wizard (HUMAN)

The init wizard is fully interactive — it requires human input at every step.

### Step 1.1 — Launch interactive init

```bash
npx codi init
```

**What to observe during the wizard:**

| # | Wizard Step | What to Look For | Status |
|---|-------------|-------------------|--------|
| 1.1a | Agent selection | Multi-select prompt listing: claude-code, cursor, codex, windsurf, cline. Select **all 5**. | [ ] |
| 1.1b | Preset selection | Shows list: minimal, balanced, strict, python-web, typescript-fullstack, security-hardened, custom. Select **balanced**. | [ ] |

| 1.1d | Confirmation | Wizard confirms and creates `.codi/` directory | [ ] |
| 1.1e | Success message | Shows summary of what was created | [ ] |

## Phase 4: Hook Lifecycle (v0.7.0 fix)

### Step 4.1 — Hooks installed after generate

```bash
cat .husky/pre-commit 2>/dev/null || echo "No husky pre-commit"
cat .husky/commit-msg 2>/dev/null || echo "No husky commit-msg"
ls .git/hooks/codi-* 2>/dev/null || echo "No standalone codi hooks"
```

| Check | Expected | Status |
|-------|----------|--------|
| Husky pre-commit has `# Codi hooks` section | Codi commands appended after existing content | [ ] |
| OR standalone hooks exist in `.git/hooks/` | `codi-secret-scan.mjs`, `codi-file-size-check.mjs` present | [ ] |
| Commit-msg validation installed | Either `.husky/commit-msg` or `.git/hooks/commit-msg` | [ ] |

### Step 4.2 — No duplicate hooks on re-generate

```bash
npx codi generate
npx codi generate
npx codi generate
grep -c "Codi hooks" .husky/pre-commit 2>/dev/null || echo "N/A"
```

| Check | Expected | Status |
|-------|----------|--------|
| Only 1 `# Codi hooks` section | Count is exactly `1`, not 2 or 3 | [ ] |
| Content not duplicated | Commands appear only once | [ ] |

### Step 4.3 — Clean preserves hooks (default)

```bash
npx codi clean --force
cat .husky/pre-commit 2>/dev/null || echo "No husky pre-commit"
ls .git/hooks/codi-* 2>/dev/null || echo "No standalone codi hooks"
```

| Check | Expected | Status |
|-------|----------|--------|
| Generated files removed | `CLAUDE.md`, `.cursorrules`, etc. are gone | [ ] |
| Hooks STILL EXIST | `# Codi hooks` section still in `.husky/pre-commit` OR `.git/hooks/codi-*` still present | [ ] |
| `.codi/` still exists | Directory preserved | [ ] |

### Step 4.4 — Clean --all removes everything including hooks

First regenerate, then clean with --all:

```bash
npx codi generate
npx codi clean --all --force
```

| Check | Expected | Status |
|-------|----------|--------|
| Generated files removed | All agent files gone | [ ] |
| Hooks removed | No `# Codi hooks` in husky, no `.git/hooks/codi-*` | [ ] |
| `.codi/` removed | Directory gone | [ ] |

**Re-initialize for remaining tests:**

