The challenge

Configuration Fragmentation

  • 5 AI coding agents, 5 different configuration formats
  • Rules duplicated across CLAUDE.md, .cursorrules, AGENTS.md, .windsurfrules, .clinerules
  • No governance — each developer configures independently
  • Configuration drift is invisible until it causes incidents

The solution

What is Codi?

A CLI that lets teams define rules, skills, agents, and flags once — then generates native config files for every AI coding agent.

Principles

Core Philosophy

Governance by Default

18 behavioral flags and 3 presets enforce best practices without requiring manual discipline.

One-Way Flow

Organization policies push down to teams and repos. Local projects cannot circumvent central governance.

Modular Artifacts

Rules, skills, agents, and commands follow identical patterns. Composable, extensible, and version-controlled.

Workflow

How It Works

1

Initialize

codi init

Interactive wizard: select agents, rules, preset. Scaffolds .codi/ directory.

2

Define

codi add rule|skill|agent

Create artifacts from 9+ built-in templates or write custom ones.

3

Generate

codi generate

Produce native config for each agent. Track state. Install hooks.

Capabilities

Main Functionalities

Governance

18 behavioral flags with 3 presets (minimal, balanced, strict). Flags can be locked at org/team level to enforce policies.

Generation

5 agent adapters produce native config files. Hash-based drift detection via state.json. Auto-backup before every generation.

Lifecycle

Full CLI: init → add → generate → status → doctor → verify → compliance. 16 commands covering the complete workflow.

Ecosystem

Skills marketplace for community sharing. Preset system for org-wide standards. CI integration with codi doctor --ci.
